Dish Import Guide

Welcome! This guide will help you prepare JSON files to import dishes into the system. Follow the format specifications below to ensure successful imports.

Fields

Field Type Requirements Example
name * String 1-100 characters "Poké Bowl"
description * String 10-500 characters "A fresh and healthy dish..."
tags * Array See valid tags below ["HEALTHY", "VEGAN"]
type * String See valid cuisine types below "JAPANESE"
imageUrl * String Max 255 chars, valid URL or path "https://example.com/image.jpg"
people * Integer 1-100 4
ingredients * Array At least 1 ingredient See ingredient format below
cookingSteps optional Array 1-50 steps See cooking steps format below

Valid Enums

Dish Tags

Choose one or more from:

DESSERT, SNACK, APPETIZER, SIDE_DISH, MAIN_COURSE, BEVERAGE, VEGETARIAN, VEGAN, GLUTEN_FREE, HEALTHY, QUICK_MEAL, COMFORT_FOOD, GRILLED, BAKED, FRIED, RAW

Cuisine Types

Select one cuisine type:

ITALIAN, FRENCH, CHINESE, JAPANESE, MEXICAN, INDIAN, AMERICAN, THAI, SPANISH, MEDITERRANEAN, MIDDLE_EASTERN, KOREAN, AFRICAN, GREEK, TURKISH, MOROCCAN

Ingredient Categories

VEGETABLES, FRUITS, DAIRY, MEAT, FISH, GRAINS, SPICES, BAKING, DRINKS, HOUSEHOLD, OTHER

Measurement Units

G, KG, ML, L, PCS, TBSP, TSP, CUP, PINCH

Ingredient Format

Each ingredient must include:

Field Type Requirements
ingredient.name * String 1-100 characters
ingredient.description String Max 250 characters (can be empty)
ingredient.category * String Valid category enum
ingredient.unit * String Valid measurement unit
quantity * Number 0.001 - 1,000,000

Cooking Steps Format (Optional)

If you want to include cooking instructions:

Field Type Requirements
orderIndex * Integer Positive number (step order)
recipeStep * String 5-255 characters

Tips for Success

Best Practices:
  • Validate your JSON using an online JSON validator before importing
  • Use descriptive names for dishes and ingredients
  • Double-check that all enum values match the valid options listed above
  • Ensure quantities are realistic and properly scaled for the number of people
  • Keep descriptions informative but concise
Common Mistakes:
  • Misspelling enum values (they are case-sensitive!)
  • Forgetting required fields like name or ingredients
  • Using invalid measurement units
  • Setting people to 0 or negative numbers
  • Description too short (minimum 10 characters) or too long

Need Help?

If you encounter any issues during import, check that:

  1. Your JSON file is properly formatted (no syntax errors)
  2. All required fields are present
  3. All values meet the validation requirements
  4. Enum values match exactly (including capitalization)
